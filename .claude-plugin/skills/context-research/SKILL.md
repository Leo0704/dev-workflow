## 执行流程

### 步骤 1: 搜索相似功能

```bash
# 使用 Glob 搜索相关文件
Glob "src/**/*.ts"  # 搜索源代码
Glob "**/*{keyword}*"  # 搜索包含关键词的文件

# 使用 Grep 搜索代码内容
Grep "关键词" --type js  # 搜索特定语言
Grep "函数名|类名"  # 搜索特定标识符
```

### 步骤 2: 分析架构

- 理解项目整体架构和模块边界
- 识别入口点和调用链
- 找出关键文件

### 步骤 3: 追踪数据流

- 追踪数据的来源和去向
- 识别外部服务依赖
- 找出可复用组件

### 步骤 4: 生成报告

使用模板: `templates/context-report.md`

```bash
# 读取模板
cat skills/context-research/templates/context-report.md

# 填充后保存到
task/{当前任务}/context-report.md
```

## 调研内容清单

1. **相似功能** - 搜索项目中已有的类似实现
2. **架构分析** - 理解项目整体架构和模块边界
3. **数据流** - 追踪数据的来源和去向
4. **外部服务** - 识别依赖的外部服务
5. **可复用组件** - 找出可以直接使用的组件

---

## 输出文件

生成报告到: `task/{任务名}/context-report.md`

## 核心原则

- 广泛搜索，不遗漏相关代码
- 深度理解，理解代码背后的设计意图
- 实用导向，关注可复用性
