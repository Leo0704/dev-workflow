## 执行流程

### 步骤 1: 依赖扫描

```bash
# 搜索直接依赖
Grep "import|require|from" --type js
Grep "useEffect|useCallback" --type tsx

# 搜索间接依赖
Grep "函数名|类名"  # 找出调用方
```

### 步骤 2: 风险评估

按以下维度分类：
- **高风险**: 核心功能、数据迁移、不可逆变更
- **中风险**: 次要功能、配置变更
- **低风险**: UI调整、日志修改

### 步骤 3: 回滚规划

评估每个变更的：
- 变更可逆性
- 回滚步骤
- 数据备份需求

### 步骤 4: 生成报告

使用模板: `templates/impact-report.md`

```bash
# 读取模板
cat skills/impact-analysis/templates/impact-report.md

# 填充后保存到
task/{当前任务}/impact-report.md
```

---

## 分析内容清单

1. **依赖扫描** - 直接/间接依赖、外部服务
2. **风险评估** - 高/中/低风险分类
3. **回滚规划** - 变更可逆性、回滚步骤
4. **缓解措施** - 针对每个风险的应对方案

---

## 输出文件

生成报告到: `task/{任务名}/impact-report.md`

---

## 核心原则

- 全面性：不遗漏潜在影响
- 优先级：先关注高风险
- 可操作性：提供具体缓解措施
