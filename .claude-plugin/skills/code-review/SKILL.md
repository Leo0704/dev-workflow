## 执行流程

### 步骤 1: 获取变更文件

```bash
# 查看本次修改的文件
git status
git diff --name-only
```

### 步骤 2: 逐文件审核

对每个修改的文件进行审核：

1. **Bug 检查**
   - 逻辑错误
   - 边界条件
   - 空指针/undefined
   - 资源泄漏

2. **质量检查**
   - 可读性
   - 复杂度
   - 重复代码
   - 函数长度

3. **规范检查**
   - 命名规范
   - 注释质量
   - 代码风格
   - 文件结构

### 步骤 3: 安全检查

- 输入验证
- SQL 注入风险
- XSS 风险
- 敏感数据处理

### 步骤 4: 生成报告

使用模板: `templates/review-report.md`

```bash
# 读取模板
cat skills/code-review/templates/review-report.md

# 填充后保存到
task/{当前任务}/review-report.md
```

---

## 审核维度

1. **Bug 检查** - 逻辑错误、边界条件、空指针
2. **质量检查** - 可读性、复杂度、重复代码
3. **规范检查** - 命名、注释、代码风格
4. **安全检查** - 输入验证、注入风险、敏感数据

---

## 输出文件

生成报告到: `task/{任务名}/review-report.md`

问题分级：
- 🔴 严重：必须修复
- 🟡 中等：建议修复
- 🟢 轻微：可选优化

---

## 核心原则

- 客观公正：基于事实和标准
- 建设性：提供具体改进建议
- 优先级：先关注严重问题
