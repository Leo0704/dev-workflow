---

## 三视角并行审核

> **必须从三个视角并行审核代码，分别关注不同角度**

```
视角 1:  Bug 审核员
─────────────────
关注点：功能性错误、逻辑漏洞、边界条件
- 空指针/未定义检查
- 边界条件处理
- 错误处理完整性
- 并发/竞态条件

视角 2:  质量审核员
─────────────────
关注点：代码质量、可维护性、性能
- 代码重复（DRY）
- 命名清晰度
- 函数长度/职责单一
- 性能问题（N+1、内存泄漏）

视角 3:  规范审核员
─────────────────
关注点：项目规范、最佳实践、安全性
- 符合 CLAUDE.md 规范
- 符合项目现有模式
- 安全漏洞（注入、XSS）
- 敏感信息泄露
```

---

## 审核执行方式

1. 顺序执行 3 个视角的审核
2. 汇总所有问题
3. 按严重程度排序
4. 输出审核报告

---

## 视角 1:  Bug 审核清单

### 功能性错误
- [ ] 空指针/未定义检查完整
- [ ] 数组/对象访问有边界检查
- [ ] 条件判断逻辑正确
- [ ] 循环终止条件正确

### 边界条件
- [ ] 空值/空数组处理
- [ ] 最大值/最小值处理
- [ ] 零值/负值处理
- [ ] 溢出/下溢保护

### 错误处理
- [ ] try-catch 覆盖关键路径
- [ ] 错误信息明确可追踪
- [ ] 失败后状态一致

### 并发/竞态
- [ ] 共享状态有锁保护
- [ ] 事务隔离级别正确
- [ ] 无死锁风险

---

## 视角 2:  质量审核清单

### DRY 原则
- [ ] 无重复代码块
- [ ] 相似逻辑抽取为函数
- [ ] 常量统一管理

### 命名清晰
- [ ] 变量名描述准确
- [ ] 函数名体现动作
- [ ] 布尔值用 is/has/should 前缀

### 函数设计
- [ ] 函数职责单一
- [ ] 参数数量合理（≤ 5 个）
- [ ] 嵌套层级 ≤ 3

### 性能
- [ ] 无 N+1 查询问题
- [ ] 大数据量有分页
- [ ] 无明显内存泄漏
- [ ] 缓存策略合理

---

## 视角 3:  规范审核清单

### 项目规范
- [ ] 符合 CLAUDE.md 约定
- [ ] 符合项目现有代码模式
- [ ] 只修改了允许修改的文件

### 安全性
- [ ] 使用参数化查询（防注入）
- [ ] 用户输入有验证
- [ ] 敏感数据不打印到日志
- [ ] 敏感字段不返回给前端
- [ ] XSS 防护（输出转义）

### 最佳实践
- [ ] 使用项目已有的工具函数
- [ ] 配置项可配置化
- [ ] 魔法数字提取为常量

## 审核结果汇总模板

```markdown
## 代码审核报告

### 审核统计
| 审核员 | 发现问题 | 高危 | 中危 | 低危 |
|--------|----------|------|------|------|
|  Bug  |   N   |  N  |  N  |  N  |
|  质量  |   N   |  N  |  N  |  N  |
|  规范  |   N   |  N  |  N  |  N  |
| **总计** |   N   |  N  |  N  |  N  |

### 高危问题（必须修复）
1. **[问题标题]** - [文件:行号]
   - 发现者: [Bug/质量/规范]
   - 说明: 问题描述
   - 修复建议: 具体修复方法

### 中危问题（建议修复）
1. **[问题标题]** - [文件:行号]
   - 发现者: xxx
   - 说明: xxx

### 低危问题（可选）
1. **[问题标题]** - [文件:行号]
   - 发现者: xxx
   - 说明: xxx

### 审核结论
- [ ] 无高危问题
- [ ] 中危问题已记录
- [ ] 代码符合项目规范

**结论:** 通过 / 需修复
```

## 使用说明

调用此技能后，将按以下流程执行：

1. 从三个视角依次审核代码
2. 汇总发现的问题
3. 输出结构化审核报告
4. 等待老板确认是否需要修复
