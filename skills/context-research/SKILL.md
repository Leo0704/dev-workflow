## 代码追踪（重要！）

> **重要：不只是搜索文件，还要追踪代码执行路径！**

### 代码追踪目标

```
┌─────────────────────────────────────────────────────────────┐
│  深度追踪相关功能的执行路径                                    │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 入口点追踪                                               │
│     - API 路由定义                                           │
│     - 事件触发点                                             │
│     - 用户交互入口                                           │
│                                                              │
│  2. 调用链追踪                                               │
│     - Handler --> Service --> Repository                    │
│     - 函数 A 调用 函数 B 调用 函数 C                          │
│     - 记录每一步的文件:行号                                   │
│                                                              │
│  3. 数据流追踪                                               │
│     - 请求参数如何传递                                        │
│     - 数据如何转换                                           │
│     - 响应如何生成                                           │
│                                                              │
│  4. 依赖关系追踪                                             │
│     - 依赖了哪些服务/模块                                     │
│     - 被哪些地方引用                                          │
│     - 配置项依赖                                              │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 追踪输出格式

```markdown
## 代码追踪报告

```

    ↓

    ↓
src/models/user.ts:10        User entity
```

### 关键文件列表
| 文件 | 作用 | 需要修改 |
|------|------|----------|
```
