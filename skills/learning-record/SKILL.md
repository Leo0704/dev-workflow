## 设计理念

**学习数据存储在插件目录，不在用户项目中**：

```
~/.claude/plugins/dev-workflow/
├── skills/
├── hooks/
└── data/                      ← 学习数据在这里
    ├── learnings.md           ← 最佳实践和经验
    └── errors.md              ← 错误和解决方案
```

---

## 执行流程

### 步骤 1: 更新工作流状态

**开始学习记录时，自动更新工作流步骤**：

```bash
echo "8" > task/$(cat task/.current-task)/.workflow-step
```

### 步骤 2: 检查待处理项

```bash
# 检查待解决的错误
cat ~/.claude/plugins/dev-workflow/data/errors.md | grep "待解决"

# 检查待确认的学习点
cat ~/.claude/plugins/dev-workflow/data/learnings.md | grep "待确认"
```

### 步骤 3: 更新错误状态

对于已解决的错误，更新状态：

```markdown
## [2026-03-05 10:30]
**命令**: `npm install`
**错误**: npm ERR!
**摘要**: ...
**状态**: 已解决
**解决**: 使用 npm cache clean --force 后重试
```

### 步骤 4: 确认学习点

查看并确认本次会话自动生成的学习点草稿：

```markdown
## [2026-03-05 14:30] 待确认学习点

**任务**: 2026-03-05-用户登录功能

**会话摘要**:
完成了完整的分析流程（5 份报告）
代码开发完成，已进入验证阶段
测试验证通过

**状态**: 待确认
```

将 `状态: 待确认` 改为 `状态: 已确认`，并补充具体的实践内容。

### 步骤 5: 提取学习点

从本次开发中提取：
- 有效的做法和模式 → 写入 learnings.md
- 常见错误和解决方案 → 更新 errors.md

### 步骤 6: 精简学习数据

如果文件超过 200 行，精简旧内容：
- 保留最近的学习
- 合并相似条目
- 删除过时内容

### 步骤 7: Git 提交建议（可选）

如果所有验证通过，提示用户提交代码：

```bash
# 查看修改
git status

# 提交
/commit

# 或提交并创建 PR
/commit-push-pr
```

## 学习文件格式

### learnings.md

```markdown
# 插件学习记录

## 最佳实践

### [2026-03-05] 使用 Agent 并行执行分析任务
**场景**: 需要同时进行需求分析、上下文调研、影响分析
**实践**: 使用 Agent 工具并行启动三个 subagent
**效果**: 减少等待时间，提高效率

## 常见问题

### 如何跳过工作流进行快速修复？
创建 .skip-workflow 文件即可跳过工作流检查。
```

### errors.md

```markdown
# 错误记录

## [2026-03-05 10:30]
**命令**: `npm install`
**错误**: npm ERR!
**摘要**: ...
**状态**: 已解决
**解决**: 使用 npm cache clean --force 后重试
```

---

## 核心原则

1. **自动记录** - 错误由钩子自动记录，学习点由钩子自动生成草稿
2. **手动精简** - 定期执行此技能精简和确认内容
3. **插件内存储** - 不在用户项目中创建文件
4. **跨项目复用** - 学习随插件走，适用于所有项目
